# 🚀 코딩테스트 스터디 Git Commit 컨벤션 가이드

팀원 간의 원활한 코드 리뷰와 효율적인 히스토리 관리를 위해 아래의 커밋 규칙을 준수해 주세요.

---

## 📌 1. 커밋 메시지 기본 양식

커밋 메시지의 제목은 한눈에 어떤 문제인지 알 수 있도록 작성합니다.

> **구조: `태그: [플랫폼] 문제이름 (난이도)`**

### 🏷️ 주요 태그(Prefix) 정의
| 태그 | 의미 | 사용 예시 |
| :--- | :--- | :--- |
| **solve** | 문제를 처음 해결하여 코드를 업로드할 때 | `solve: [PG] 타겟 넘버 (Lv.2)` |
| **fix** | 오답 수정이나 런타임 에러를 해결했을 때 | `fix: [BOJ] 1, 2, 3 더하기 (오답 수정)` |
| **refactor** | 정답은 맞았으나 로직을 개선하거나 가독성을 높일 때 | `refactor: [PG] 교점에 별 만들기 (메모리 최적화)` |
| **docs** | README.md 등 문서 파일을 수정할 때 | `docs: 스터디 규칙 및 일정 업데이트` |
| **chore** | .gitignore 설정 등 기타 환경 설정 변경 시 | `chore: 프로젝트 모듈 구조 변경` |

---

## 📌 2. 상세 설명(Body) 권장 사항 (선택)

코드 리뷰 시 팀원들이 로직을 빠르게 이해할 수 있도록 본문에 아래 내용을 포함하는 것을 권장합니다.
- **유형:** (예: DFS/BFS, 구현, 그리디)
- **시간 복잡도:** (예: O(N^2))
- **핵심 로직:** 간단한 설명 (1~3줄)

---

## 📌 3. 실제 커밋 내역 예시 (Example)

우리 레포지토리에 쌓일 커밋 히스토리는 다음과 같은 모습이 됩니다.

### ✅ 예시 1: 신규 문제 풀이 (심플형)
```text
solve: [프로그래머스] 교점에 별 만들기 (Lv.2)

```

### ✅ 예시 2: 상세 설명 포함 (권장형)

```text
solve: [프로그래머스] 교점에 별 만들기 (Lv.2)

- 유형: 구현, 기하
- 시간 복잡도: O(N^2) (N은 직선의 개수)
- 핵심 로직: 
  1. 두 직선의 교점 공식으로 정수 좌표 추출
  2. 좌표들의 최소/최대값을 구해 별을 그릴 도화지 크기 결정

```

### ✅ 예시 3: 코드 개선 (리팩토링)

```text
refactor: [프로그래머스] 교점에 별 만들기 (Lv.2)

- 기존의 2차원 배열 방식에서 메모리 절약을 위해 StringBuilder 방식으로 변경

```

## 💡 팀원 필독 사항

1. **커밋 시점:** 각자 정해진 스터디 시간 전까지 커밋을 완료해 주세요.
2. **PR 제목:** `[이름] n주차 문제 풀이 제출 (n개)` 형식으로 생성해 주세요.
3. **코드 리뷰:** 서로의 커밋 내역을 확인하고 궁금한 점은 코멘트로 남겨주세요!
